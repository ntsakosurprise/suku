---

## Reporting a Vulnerability

If you discover a security vulnerability in SUKU, please report it responsibly:

1. **Email:** security@sukujs.org  
2. **PGP Key:** [Optional PGP key link]  
3. **Do not** create a public issue for security bugs.

Please include:
- A detailed description of the issue
- Steps to reproduce
- Environment details (OS, Node/browser version, SUKU version)
- Any suggested fixes if available

We will respond within **48 hours** and work to release a patch promptly.

---

## Security Best Practices

- Always update to the latest SUKU version.
- Validate and sanitize all external inputs.
- When using plugins, ensure they come from trusted sources.
- Avoid running SUKU in environments with elevated privileges unless necessary.

---

## Security Updates

Security patches will be released via:
- GitHub releases
- NPM package updates
- [Optional] Security advisories mailing list

---

## Reporting Policy

SUKU follows responsible disclosure guidelines:
- Researchers will be acknowledged unless they request anonymity.
- Fixes will be coordinated to avoid exposing users before a patch is available.
- Critical issues may trigger a minor or patch release immediately.

---

Thank you for helping keep SUKU safe!
